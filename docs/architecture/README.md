# Architecture

`docs/architecture/` contains architecture briefs and supporting documentation that explain how Djoko Studio is structured and why.

Use this folder for:

- conceptual architecture briefs
- system boundaries and responsibilities
- data flow explanations
- operational and reliability notes
- supporting diagrams and future architecture references

Architecture briefs describe the system structure, reasoning, and open questions.

ADRs record decisions that are accepted, along with their context and consequences.
